# GenAI Medical Chatbot – Multi-Agent System

A Generative AI-powered medical information chatbot built using a multi-agent architecture (Planner → Executor → Verifier). The system provides safe, non-diagnostic, educational responses for common and mild medical conditions.

Disclaimer:  
This application is intended for educational purposes only and does not provide medical diagnosis, treatment, prescription, or emergency medical advice.

# System Architecture

The application follows a multi-agent workflow to ensure structured execution and safer response generation.

## 1. Planner Agent

Responsibilities:

- Understands user intent
- Determines execution flow
- Decides whether to use:
  - LLM generation
  - FAISS retrieval
  - External APIs

---

## 2. Executor Agent

Responsibilities:

- Executes the plan generated by the Planner
- Calls integrated tools such as:
  - FLAN-T5 generation
  - FAISS vector retrieval
  - Hugging Face APIs

---

## 3. Verifier Agent

Responsibilities:

- Validates generated responses for:
  - Safety
  - Clarity
  - Neutral medical tone
  - Non-diagnostic behavior
- Formats final output for UI display

# Known Limitations

- Not intended for emergency or critical medical conditions
- No access to real-time hospital or clinical databases
- Output quality depends on model knowledge and dataset quality
- FAISS retrieval accuracy depends on embedding quality
- System intentionally limited to mild medical conditions
